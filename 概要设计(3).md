## 1 引言

### 1.1 编写目的

本文档是鳕鱼音乐在线生成系统概要设计阶段的工作成果。其目的在于将软件需求转换为系统的高层设计，定义系统总体结构、模块划分、接口设计、运行设计及数据结构设计，为后续详细设计、编码实现、测试及维护提供依据。

预期读者包括：项目经理、系统架构师、开发工程师、测试工程师及运维人员。

---

### 1.2 背景

软件系统名称：鳕鱼音乐在线生成系统
开发者：[鳕鱼音乐开发团队] 
目标用户：音乐创作者、视频制作者、广告从业者及普通音乐爱好者  

系统部署于公有云环境，通过 Internet 提供服务。系统通过 HTTPS 协议调用第三方 AI 音乐生成接口（ Suno API、Udio API），并与支付平台及对象存储服务进行集成。

## 2 总体设计

### 2.1 需求规定

本系统基于《MGS 软件需求规格说明书》中的功能需求与非功能需求进行设计，主要需求如下：

#### 2.1.1 功能需求

系统应具备如下功能：

1. 用户账户管理：支持注册、登录、身份认证及密码找回；
2. 音乐创作：支持基于文本描述进行音乐创作，并支持风格、情绪、乐器、时长等参数配置，以及任务提交与状态管理；
3. 作品管理：支持作品浏览、在线试听、下载、收藏、删除及分享；
4. 订单管理：支持订单创建、订单查询、支付记录管理；
5. 计费管理：支持积分消耗、套餐订阅、配额控制与消费统计；
6. 通知管理：支持生成结果通知、订单通知及系统消息推送；
7. 后台管理：支持用户管理、运营监控、系统配置及数据统计。

#### 2.1.2 性能需求

1. 页面首屏加载时间不超过 2 秒；  
2. 生成请求提交响应时间不超过 500ms；  
3. 支持不少于 10 并发用户；  
4. 音频下载速度不低于 1MB/s。

### 2.3 基本设计概念

#### 2.3.1 设计思想

系统采用如下设计原则：

1. 前后端分离架构；  
2. 微服务模块化设计；  
3. 异步任务处理机制；  
4. API 网关统一入口控制；  
5. 外部接口通过适配层解耦。

---

### 2.4 设计约束

系统设计需满足以下约束：

1. 必须依赖第三方 AI 音乐生成接口；  
2. 系统需部署于公有云环境；  
3. 必须符合网络安全等级保护要求；  
4. 支付流程必须符合第三方支付规范；  
5. 所有外部通信必须使用 HTTPS。

### 3.1 用户接口

- **Web 端**：响应式布局，适配桌面端与移动端
- **交互规范**：遵循 Ant Design / Element Plus 设计规范
- **主要页面**：
  - 首页（功能介绍 + 快捷生成入口）
  - 生成页（参数配置表单 + 实时预览）
  - 作品库页（列表/网格视图 + 筛选排序）

  - 管理后台（Dashboard、数据表格、配置面板）

### 3.2 外部接口

#### 3.2.1 AI 音乐创作接口

系统使用第三方 AI 音乐创作服务实现歌曲生成。

Suno API：

接口地址：
POST https://api.sunoapi.org/api/v1/generate

主要参数：

- prompt：歌曲描述文本
- style：音乐风格
- instrumental：是否纯音乐
- duration：歌曲时长

返回：

- task_id
- status
- audio_url

Udio API：

接口地址：

POST https://api.udio.com/v1/music/generate

主要参数：

- prompt
- genre
- mood
- lyrics

返回：

- generation_id
- status
- result_url

#### 3.2.2 支付接口

| 项目     | 说明                                                     |
| :------- | :------------------------------------------------------- |
| 微信支付 | JSAPI / Native / H5 支付，遵循微信支付 V3 协议           |
| 支付宝   | 手机网站支付 / 电脑网站支付，遵循支付宝开放平台协议      |
| 回调地址 | `https://api.mgs.example.com/webhooks/payment/{channel}` |

#### 3.2.3 对象存储接口

| 项目     | 说明                                        |
| :------- | :------------------------------------------ |
| 服务     | 阿里云 OSS / AWS S3                         |
| 上传方式 | 服务端预签名 URL（Presigned URL），前端直传 |
| 存储路径 | `audio/{user_id}/{date}/{uuid}.{ext}`       |
| CDN 加速 | 通过自定义域名 + CDN 分发音频文件           |

## 4 运行设计

### 4.1 运行模块组合

| 运行场景     | 参与模块组合                         | 说明                                |
| :----------- | :----------------------------------- | :---------------------------------- |
| 用户注册登录 | M01 + 数据层                         | 完成身份认证与会话建立              |
| 音乐生成     | M02 + M03 + M04 + M06 + M08 + 数据层 | 校验配额→提交任务→异步生成→结果通知 |
| 作品浏览下载 | M05 + 数据层 + OSS/CDN               | 查询元数据→返回 CDN 播放地址        |
| 支付订阅     | M06 + 外部支付 API + 数据层          | 创建订单→调起支付→回调处理→更新配额 |
| 系统监控     | M07 + Prometheus/Grafana             | 采集指标→可视化展示                 |

### 4.2 运行流程

音乐生成流程如下：

1. 用户提交生成请求；
2. 网关完成认证与限流；
3. 生成服务校验参数；
4. 写入消息队列；
5. 异步调用外部 API；
6. 获取结果并存储；
7. 通知用户。

------

### 4.3 运行控制

系统通过以下方式进行控制：

1. API 网关统一鉴权；
2. Redis 实现限流；
3. 消息队列控制任务优先级；
4. 熔断机制防止外部依赖失效。

- **PostgreSQL**：主从架构，主库负责写，从库负责读，通过 PgPool-II 或云数据库代理实现读写分离
- **Redis**：
  - DB0：用户会话缓存（JWT Blacklist、Rate Limit 计数）
  - DB1：生成任务状态缓存（加速状态查询）
  - DB2：热点音频元数据缓存
- **OSS**：音频文件按 `audio/{user_id}/{yyyyMM}/{uuid}.mp3` 路径存储，生命周期策略：30 天后转低频存储，1 年后归档

## 6 系统出错处理设计

### 6.1 出错信息

| 错误类别      | 错误码 | 错误信息                 | 处理策略                             |
| :------------ | :----- | :----------------------- | :----------------------------------- |
| 参数校验失败  | 400001 | 请求参数非法             | 返回具体字段错误，前端表单高亮       |
| 认证失败      | 401001 | Token 无效或过期         | 引导用户重新登录                     |
| 配额不足      | 403001 | 今日生成次数已用完       | 引导用户购买套餐                     |
| 外部 API 限流 | 429001 | 生成服务繁忙，请稍后重试 | 指数退避重试，最大重试 3 次          |
| 外部 API 故障 | 503001 | 第三方服务暂时不可用     | 熔断降级，返回友好提示，记录日志告警 |
| 任务超时      | 504001 | 生成任务超时             | 标记失败，自动退款或返还积分         |
| 系统内部错误  | 500001 | 服务器内部错误           | 记录详细堆栈，不暴露内部信息给客户端 |

### 6.2 补救措施

- **任务失败重试**：消息队列消费端配置 `maxRetries=3`，失败消息进入死信队列（DLQ），人工介入或定时补偿
- **数据一致性**：采用最终一致性方案，支付回调与配额更新通过事务消息保证
- **备份恢复**：PostgreSQL 每日自动全量备份 + WAL 归档，RPO ≤ 1 小时；OSS 开启跨区域复制
- **降级策略**：当外部 AI-API 不可用时，暂停生成入口，保留已有作品的浏览下载功能

### 6.3 系统维护设计

- **日志规范**：采用结构化日志（JSON），统一收集至 ELK（Elasticsearch + Logstash + Kibana）或云日志服务
- **监控告警**：Prometheus + Grafana 监控 QPS、延迟、错误率、队列堆积深度；告警通道：钉钉/企业微信/邮件
- **配置中心**：使用 Nacos / Apollo 集中管理配置，支持动态刷新（如调整限流阈值、开关功能）
- **版本发布**：基于 Kubernetes 的滚动更新（Rolling Update），配合健康检查确保零停机发布

## 7 安全性设计

系统安全策略如下：

1. **传输安全**：全站 HTTPS（TLS 1.3），HSTS 头部加固
2. **数据安全**：用户密码 bcrypt 加盐哈希；敏感配置（API Key、数据库密码）存入 Vault/阿里云 KMS
3. **API 安全**：防重放攻击（请求时间戳 + Nonce 校验）、防 SQL 注入（ORM 参数化查询）、防 XSS（前端转义 + CSP 策略）
4. **合规性**：遵循《网络安全法》《个人信息保护法》，用户数据匿名化处理，提供数据导出与删除功能
